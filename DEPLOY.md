# Как запустить бота (пошагово, без опыта программирования)

Все файлы уже готовы, в них ничего менять не нужно. Ваша задача — один раз разместить их
на GitHub и включить расписание. Дальше всё будет работать само.

**Важно про Kwork:** этот источник рендерится через браузер (Playwright), поэтому первый
запуск займёт на 1-2 минуты дольше — устанавливается headless-Chromium. Я не смогла
протестировать разбор именно Kwork вживую (моя песочница не имеет туда доступа), так что
после первого реального запуска пришлите мне лог из вкладки Actions — если с Kwork
что-то не так, поправлю парсер по факту, не трогая остальные источники.

## Шаг 1. Узнать CHAT_ID вашего бота

1. Убедитесь, что вы уже написали своему боту хотя бы одно сообщение в Telegram (например, «привет») — без этого бот не сможет вам ответить.
2. Откройте в браузере (на компьютере или телефоне) вот такую ссылку, подставив свой токен вместо `<ТОКЕН>`:
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
   Пример с вашим токеном:
   `https://api.telegram.org/bot8792520684:AAGfEo9_duqcWFSEX-xPBZYsbnF5DEE7Tcw/getUpdates`
3. На странице появится текст (JSON). Найдите там кусок `"chat":{"id":123456789,...` — число после `"id":` и есть ваш CHAT_ID. Скопируйте его (это просто число, без кавычек).

Если страница пустая — напишите боту ещё раз сообщение и обновите страницу.

## Шаг 2. Создать аккаунт GitHub (если его ещё нет)

1. Зайдите на [github.com](https://github.com) → «Sign up» → следуйте подсказкам (email, пароль). Это бесплатно.

## Шаг 3. Создать репозиторий (папку проекта)

1. Нажмите зелёную кнопку «New» (или значок «+» → «New repository»).
2. Название — любое, например `vacancy-bot`.
3. Выберите **«Public»** (это важно — см. пояснение ниже, кнопка «обновить сейчас» без этого может перестать работать в какой-то момент из-за лимита бесплатных минут).
4. Нажмите «Create repository».

Почему Public: чтобы кнопка «Обновить сейчас» в Telegram могла проверяться раз в 5 минут, репозиторию нужно часто «просыпаться». В приватных репозиториях это ограничено бесплатным лимитом минут в месяц, в публичных — лимита нет. Токен бота (`BOT_TOKEN`) и `CHAT_ID` при этом остаются полностью скрыты в любом случае — они хранятся в Secrets отдельно от кода и не видны никому, даже в публичном репозитории. Видно будет только сам код (список каналов, ключевые слова, шаблон отклика).

## Шаг 4. Загрузить файлы

1. На странице репозитория нажмите «Add file» → «Upload files».
2. Перетащите в окно браузера всю папку `vacancy_bot` целиком (вместе с вложенной папкой `.github`) — так, как она у вас есть после скачивания.
   - Если GitHub не даёт загрузить папку с подпапками через браузер, загрузите файлы по отдельности, вручную создав такую же структуру:
     ```
     vacancy_bot/main.py
     vacancy_bot/check_command.py
     vacancy_bot/requirements.txt
     vacancy_bot/seen.json
     vacancy_bot/sources.json
     vacancy_bot/update_offset.json
     .github/workflows/vacancy_digest.yml
     .github/workflows/check_command.yml
     ```
3. Нажмите «Commit changes» внизу страницы.

## Шаг 5. Добавить секреты (токен и chat_id)

Секреты — это то место, где токен хранится безопасно и не виден в коде.

1. В репозитории откройте «Settings» → слева «Secrets and variables» → «Actions».
2. Нажмите «New repository secret».
   - Name: `BOT_TOKEN`, Value: ваш токен бота (тот, что от @BotFather) → «Add secret».
3. Ещё раз «New repository secret».
   - Name: `CHAT_ID`, Value: число из Шага 1 → «Add secret».

(`GITHUB_TOKEN`, нужный для кнопки «обновить сейчас», добавлять не нужно — GitHub создаёт и подставляет его автоматически в каждом запуске.)

## Шаг 6. Разрешить Actions запускать другие Actions (для кнопки «обновить сейчас»)

Без этого шага кнопка «Обновить сейчас» будет получать сообщение «Принято, обновляю…», но реального запуска сборки не будет — GitHub по умолчанию не даёт автоматическому токену запускать workflow.

1. «Settings» → слева «Actions» → «General».
2. Прокрутите вниз до «Workflow permissions».
3. Выберите «Read and write permissions».
4. Нажмите «Save».

## Шаг 7. Включить и проверить расписание

1. Откройте вкладку «Actions» вверху репозитория.
2. Если появится предложение включить Actions — нажмите «I understand my workflows, go ahead and enable them».
3. Слева выберите workflow «Vacancy digest».
4. Нажмите «Run workflow» → «Run workflow» ещё раз (для ручного теста, не дожидаясь расписания).
5. Подождите 1–2 минуты, обновите страницу — должна появиться зелёная галочka (успех) или красный крестик (ошибка, тогда пришлите мне текст ошибки — открывается по клику на прогон).
6. Проверьте Telegram — бот должен был прислать сообщение (подборку вакансий или «новых вакансий не нашлось») и под ним должна появиться кнопка «🔄 Обновить сейчас».

Дальше ничего делать не нужно — бот будет сам просыпаться в 10:00 и 18:00 по Москве, а также каждые 5 минут незаметно проверять, не нажали ли вы кнопку.

## Как работает кнопка «Обновить сейчас»

Нажмите кнопку внизу чата (или напишите боту «обновить») в любое время — бот пришлёт «Принято, обновляю…», а затем в течение примерно 1–6 минут запустит полную проверку всех источников и пришлёт новые вакансии (или сообщение, что новых нет). Задержка связана с тем, что у бота нет постоянно включённого сервера — проверка команды идёт по расписанию раз в 5 минут, а не мгновенно.

## Если что-то не работает

Пришлите мне текст ошибки из вкладки Actions (открыть неудачный прогон → развернуть шаг с ошибкой) — я поправлю код.
